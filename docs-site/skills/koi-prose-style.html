koi-prose-style

Любой текст в интерфейсе

Привести видимый текст проекта к естественному языку и записать только после проверки стиля.

Что делает

Проверяет и переписывает человекочитаемый текст: заголовки узлов и карточек, описания, исследовательские вопросы и курируемые заметки. Цикл: черновик → ревью стиля → правка → снова ревью. В файл — только после «пройдено». Заголовки короткие; детали — в описаниях.

Зачем

Чтобы карту исследования и ответы в интерфейсе понимал человек без внутреннего глоссария проекта, без каши языков и без технических идентификаторов на виду.

Когда вызывать

  • Добавляете или правите карточки и узлы дерева
  • Формулируете вопрос и ответ после эксперимента
  • Пишете любой текст, который увидят в интерфейсе без открытия полного отчёта

Как вызвать

Попросите переписать конкретные заголовки или фрагмент по правилам стиля. Для строгой проверки AI-помощник гоняет цикл до «пройдено».

Пример фразы AI-помощнику в редакторе:

Перепиши заголовки узлов в файле проекта
по правилам стиля: короткий естественный язык, детали — в описаниях.

Схема работы

Схема — замкнутый цикл качества текста. Пока ревью не сказало «пройдено», запись в файл запрещена: только правка и повторная проверка.

flowchart LR
  A[Черновик] --> B[Ревью стиля]
  B --> C{Пройдено?}
  C -->|нет| D[Переписать]
  D --> B
  C -->|да| E[Запись в файл]
