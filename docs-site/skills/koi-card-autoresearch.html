Автоисследование по карточке

«Системное исследование» / многочасовой прогон

Длинный прогон одной карточки с тремя ролями: Руководитель, Исследователь и Дебаггер — поверх выполнения карточки.

Что делает

Оркестрация долгого эксперимента по одной карточке канбана. Руководитель один раз переносит карточку в работу, настраивает живое окно и state-файл. Исследователь запускает job (через project-скилл, если он есть), следит по расписанию и отмечает подзадачи. Дебаггер только диагностирует сбои и пишет рекомендации — код и перезапуск остаются за Исследователем. Канбан, отчёт и колонка «готово» ведутся по сценарию выполнения карточки.

Зачем

Чтобы многочасовой или remote-прогон не разваливался на «запустил и забыл»: есть кто стартует, кто крутит job и смотрит пульс, кто только разбирает ошибки — без смешения ролей в одном хаотичном чате.

Когда вызывать

  • Просьба «проведи системное исследование / автоисследование по карточке»
  • Обучение или remote job на часы–сутки: нужен watch и плановый triage сбоев
  • В workspace есть project-скилл запуска (например для verl) — нужен общий протокол ролей поверх него

Как вызвать

Скажите «проведи системное исследование по карточке …» или «автоисследование по карточке …». Назовите проект и id/заголовок карточки. Если рядом с кодом есть свой скилл запуска — упомяните его; роли и cadence всё равно из этого сценария, а канбан — из выполнения карточки.

Пример фразы AI-помощнику в редакторе:

Проведи системное исследование по карточке <id> в проекте <проект>:
роли руководитель → исследователь → дебаггер, с мониторингом по ходу.

Схема работы

Схема — три роли по времени. Руководитель работает один раз в начале. Исследователь и Дебаггер идут параллельно: Исследователь чаще смотрит пульс job, Дебаггер — плановый triage и внеплановый вызов. Финиш — отчёт и колонка «готово».

flowchart TD
  U[Запрос пользователя] --> M[Руководитель: в работу + live + state]
  M --> R[Исследователь: старт job]
  R --> W[Watch: 1 мин × 20, потом 20 мин]
  R --> D[Дебаггер: каждые 10 мин]
  W -->|сбой| I[Вызов дебаггера]
  I --> D
  D -->|рекомендация в state| W
  W -->|готово| F[Отчёт + готово + вывод]
