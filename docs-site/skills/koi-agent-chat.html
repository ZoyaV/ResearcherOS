Вопрос из панели «Спросить агента»

Ответить на вопрос из панели ResearcherOS: сначала база выводов, при необходимости — отчёты.

Что делает

Забирает вопрос из очереди интерфейса. Если он хорошо совпадает с уже записанными выводами — отвечает быстро. Иначе читает нужные отчёты и кладёт ответ так, чтобы он появился в панели чата. Режим работы настраивается: постоянный Inbox-чат, хуки редактора или фоновый ключ API.

Зачем

Чтобы вопросы к проекту задавались из интерфейса ResearcherOS и получали ответ из уже накопленных выводов и отчётов, а не терялись в обычном чате без связи с картой исследования.

Когда вызывать

  • В панели «Спросить агента» появился новый вопрос
  • При старте сессии в очереди есть необработанные вопросы
  • Нужен ответ с опорой на базу выводов, а не на догадку

Как вызвать

Попросите проверить очередь вопросов из интерфейса и ответить по этому сценарию. Для быстрого режима настройте Inbox-чат или хуки по инструкции в документации агентного чата.

Пример фразы AI-помощнику в редакторе:

Проверь очередь вопросов из интерфейса и ответь
на необработанные по сценарию чата ResearcherOS.

Схема работы

Схема — развилка по наличию готового вывода. Сначала проверка базы ответов; чтение отчётов — только если готового совпадения нет. Ответ всегда возвращается в очередь, чтобы панель чата его показала.

flowchart TD
  A[Вопрос из интерфейса] --> B{Есть готовый вывод?}
  B -->|да| C[Быстрый ответ]
  B -->|нет| D[Прочитать отчёты]
  D --> E[Ответ в очередь]
  E --> F[Показать в панели чата]
