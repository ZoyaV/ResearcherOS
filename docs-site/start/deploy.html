Проект уже на ResearchOS — только запустить

Этот путь — когда дерево гипотез и материалы исследования уже есть у вас или у команды. Нужно понять, где они живут, поставить движок и подтянуть данные рядом с кодом.

Как это устроено

Репозиторий ResearchOS — это движок: интерфейс (client), сервер, API и набор сценариев для AI-помощника. Данные ваших проектов внутри него не хранятся.

При запуске сервер смотрит на папки на том же уровне, что и ReseachOS/ (и на пути из KOI_SCAN_ROOTS, если заданы). В каждой такой папке он ищет koi-structure/ — там дерево гипотез, доска, отчёты и выводы. По этим файлам интерфейс строит граф. 

workspace/
├── ReseachOS/              ← движок (client + server + API + skills)
├── TalkingHeads/           ← ваш код-репозиторий (пример)
│   ├── … код …
│   └── koi-structure/      ← материалы исследования (если подтянуты)
├── verl-agent-craftext/    ← другой проект — та же схема
└── OtherRepo/

Где лежит koi-structure в git

В репозитории с кодом (TalkingHeads, verl-agent-craftext, любой другой) папка koi-structure/ обычно живёт не в ветке с кодом, а в отдельной «ветке-сироте» (orphan-ветке) koi/research.

У неё своя история: в коммитах только материалы исследования. Так люди могут работать в разных ветках кода (main, feature-ветки) и всё равно синхронизировать одну и ту же карту исследования. На ветке с кодом koi-structure/ часто в .gitignore — локально видна, в PR с кодом не попадает.

Что будет после обычного clone

По умолчанию в папке с кодом скачивается ветка main (или другая code-ветка). Тогда структура выглядит так — без материалов исследования: 

ReseachOS/
TalkingHeads/          ← ветка main
  … talking_heads_code …

Чтобы сервер увидел проект, рядом с кодом должна появиться koi-structure/ из ветки koi/research: 

ReseachOS/
TalkingHeads/          ← ветка main (код)
  … talking_heads_code …
  koi-structure/       ← содержимое ветки koi/research

Попросите AI-помощника установить / подтянуть ветку koi/research внутри папки с кодом того репозитория, который хотите видеть в интерфейсе. Либо сделайте это командами ниже.

Пошагово

  1. Клон движка и зависимости 
    git clone git@github.com:ZoyaV/ReseacherOS.git ReseachOS
cd ReseachOS
conda create -n researchos python=3.11 -y && conda activate researchos
# или: python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
  2. Репозиторий с кодом — sibling рядом с ReseachOS

    Клонируйте (или положите) code-репо на том же уровне, что и движок. Имена папок любые — важно соседство и наличие koi-structure/ после шага 3. 

    cd ..   # родитель ReseachOS/
git clone <url-вашего-code-репо> TalkingHeads
# или: verl-agent-craftext, OtherRepo, …
  3. Подтянуть материалы исследования (ветка koi/research)

    Если ветка уже есть на remote, укажите её в шапке project.md (когда файлы появятся локально) и выполните pull через CLI движка — или попросите AI-помощника сделать то же самое. 

    # пример фразы AI-помощнику:
# В репозитории TalkingHeads (рядом с ReseachOS) установи / подтяни
# orphan-ветку koi/research так, чтобы рядом с кодом появилась koi-structure/,
# а текущая code-ветка осталась main. Не меняй git config.

# или вручную через CLI движка (когда project.md уже указывает sync-ветку):
cd ReseachOS
python -m koi.projects.sync_cli pull --project-id <your-project-id>

# в koi-structure/project.md должно быть примерно:
# git_repo: true
# git_sync_branch: koi/research

    Если ветки ещё нет — это уже путь «впервые подключаю свой код» (онбординг + создание ветки).

  4. Запуск 
    cd ReseachOS
./scripts/koi-serve.sh start
./scripts/koi-serve.sh status
# интерфейс  http://127.0.0.1:8080
# API        http://127.0.0.1:8010/docs

    В сайдбаре должен появиться проект: сервер нашёл соседнюю папку с koi-structure/project.md и построил граф.

Дальше синхронизация — кнопка Sync в интерфейсе или сценарий синхронизации. Секреты — в .env (см. .env.example), не в репозитории.
