Впервые подключаю свой код

Отличие от «уже на ResearchOS»: код эксперимента есть, а карты исследования ещё нет. AI-помощник по сценарию подключения проекта соберёт дерево гипотез и запишет материалы рядом с кодом.

  1. Поставьте ResearchOS рядом с проектом 
    git clone git@github.com:ZoyaV/ReseacherOS.git ReseachOS
# рядом:
# workspace/ReseachOS/
# workspace/my_experiment/
  2. Запустите ResearchOS 
    cd ReseachOS
./scripts/koi-serve.sh start
# интерфейс: http://127.0.0.1:8080
  3. Попросите AI-помощника подключить репозиторий

    Вставьте в Cursor или Claude Code:

    Подключи ../my_experiment к ResearchOS.

Следуй agents/skills/koi-project-onboard/SKILL.md до конца,
включая отдельную ветку для материалов исследования (обычно koi/research).
  4. Дальше в интерфейсе

    После записи project.md появятся дерево и доска прогонов. Детали эксперимента — сценарий проектирования эксперимента.

Зачем отдельная ветка для исследования

В одном репозитории живут две разные вещи: код экспериментов и материалы исследования (дерево гипотез, доска прогонов, отчёты, выводы). Если коммитить всё в одну ветку с кодом, pull request’ы и ревью кода захламляются правками карты и отчётов.

ResearchOS поэтому использует приём с отдельной «веткой-сиротой» (orphan branch): у неё нет общей истории с веткой кода. В эту ветку (по умолчанию koi/research) уезжает только папка koi-structure/. На ветке с кодом эта папка добавляется в .gitignore — локально вы её видите и правите, а в коммиты кода она не попадает.

Как это выглядит в одном репозитории проекта:

ветка main (или develop)     →  код, скрипты, конфиги
                                koi-structure/ в .gitignore

ветка koi/research (orphan)  →  только koi-structure/
                                дерево, отчёты, research.json

AI-помощник при подключении обычно сам создаёт ветку, делает первый push и прописывает имя ветки в шапке project.md (git_sync_branch). Дальше — кнопка Sync в интерфейсе или сценарий синхронизации: «забрать» чужие правки исследования и «отправить» свои.

Если ветка уже есть на remote (коллега или CI уже создали koi/research) — не создавайте вторую. Укажите то же имя в git_sync_branch и выполните pull: 

cd ReseachOS
python -m koi.projects.sync_cli pull --project-id <id-проекта>
