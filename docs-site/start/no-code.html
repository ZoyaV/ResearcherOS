Начинаю с нуля — кода ещё нет

Это не «подключение готового кода». Сначала заводим пустой проект и карту гипотез — код эксперимента появится позже в projectcode/.

В интерфейсе

  1. Клонируйте и запустите ResearchOS 
    git clone git@github.com:ZoyaV/ReseacherOS.git ReseachOS
cd ReseachOS
./scripts/koi-serve.sh start
# http://127.0.0.1:8080
  2. В списке проектов нажмите + Новый проект: название, описание, короткий латинский ярлык папки и программу.
  3. На карте появится проблема — добавляйте причины, гипотезы и методы через «+», а прогоны — на доске метода.

С помощником в редакторе

Промпт:

Создай новый проект ResearchOS с нуля.

1. Спроси меня о проблеме, области и том, какие эксперименты реалистичны.
2. Предложи дерево: проблема → причины → гипотезы → методы.
3. Спроси короткий латинский ярлык папки и создай её рядом с ReseachOS/
   с материалами исследования и папкой projectcode/.
4. Пиши понятным языком. Не коммить без моей явной просьбы.

Пошаговый учебный проход — getting-started. Демо: ветка test_project → папка bicycle_problem.
