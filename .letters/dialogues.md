# Диалоги о ResearchOS (симуляция по persona-скиллам)

Каждому представлены 10 фич (см. `researchos-features.md`). Ниже — сжатые диалоги и выводы для писем.

---

## 1. Виталий Баган

**Презентация:** F1–F10 кратко.

**Б:** «Проблема, которую я вижу в лабораториях, — не нехватка идей, а потеря контекста. Аспирант уходит — уходит полгода работы. Что из вашего списка это решает?»

**М:** F4 — автоматическая база знаний из вердиктов и инсайтов; F8 — программы объединяют проекты на стратегическом уровне.

**Б:** «База без LLM — это важно. Не хочу, чтобы „знание" галлюцинировалось. Как онбординг?»

**М:** Новый человек открывает KNOWLEDGE.md — видит подтверждённые/опровергнутые гипотезы, числа, ссылки на отчёты.

**Б:** «ИИ-агенты — с осторожностью. Вердикт из правила, не из модели?»

**М:** Да, F2: правило решения до прогона; ingest отбраковывает отчёт без §5.1.

**Б:** «Open Source для университета — правильно. Свяжите с инфраструктурой сопровождения НИР. Пилот — 2–3 лаборатории стратпроекта, не весь вуз сразу.»

**Приоритеты для письма:** F4, F8, F10, F2.

## 2. Николай Кондратюк

**Презентация:** F1–F10.

**К:** «Я каждый месяц пересчитываю то, что уже считал, потому что прошлый прогон „где-то был". У вас это как?»

**М:** F4 + F5: карточка канбана → отчёт → инсайты в research.json → KNOWLEDGE.md. F3: всё в git.

**К:** «А воспроизводимость? В MD-симуляциях seed, версия LAMMPS, параметры кластера — без этого формальность.»

**М:** Шаблон experiment-report.md, §3 подзадачи с артефактами; koi-report-review проверяет.

**К:** «Предрегистрация — это то, что отличает эксперимент от „погоняли и посмотрели". F2 — главное.»

**М:** Правило в cause/method до running; вердикт — подстановка чисел.

**К:** «ИИ пусть генерирует input-файлы. Вердикт — только человек/числа. Поддержу, если покажете на MD-лаборатории.»

**Приоритеты для письма:** F2, F4, F5, F3.

---

## 3. Андрей Райгородский

**Презентация:** F1–F10.

**Р:** «ИИИ создавался как платформа для сложных задач. Что ResearchOS даёт поверх Cursor и Jupyter?»

**М:** F1+F6: не код ради кода, а цепочка гипотеза→эксперимент→вердикт→БЗ. F7: несколько репо — одно дерево.

**Р:** «Composite — интересно. Лаборатория A — одна ветка, лаборатория B — другая, общие cause-узлы. Это наш реальный режим.»

**М:** composite_id, read-time merge, write routing по project_id.

**Р:** «Open Source — стратегически. Методология разъедется быстрее закрытого продукта. Пилот в ИИИ — да, с метриками через полгода.»

**Приоритеты для письма:** F6, F7, F10, F1, пилот ИИИ.

## 4. Юрий Хворост

**Презентация:** F1–F10.

**Х:** «Мне не нужен ещё один SaaS. Что если завтра санкции, смена API, закрытие сервиса?»

**М:** F3: markdown в git, без БД. F9: Cursor, Claude Code, inbox — смена бэкенда без смены данных.

**Х:** «Vendor-neutral — ключевое. Отечественные модели появляются каждый квартал. Платформа должна их подхватывать за дни, не месяцы.»

**М:** Skills и AGENTS.md не привязаны к провайдеру; GET /agent/backends.

**Х:** «Open Source + git — единственный способ масштабировать на вузы без команды интеграторов. Цифровое небо могло бы быть вторым пилотом после ИИИ.»

**Приоритеты для письма:** F9, F3, F10, F7.

---

## 5. Азамат Жилоков

**Презентация:** F1–F10.

**Ж:** «Нейросети у нас уже в ежедневной работе — гипотезы, литература, код. Что ResearchOS добавляет?»

**М:** F6: агент не в чате, а в канбане — карточка, running, отчёт, done, ingest. F5: live view активности.

**Ж:** «Ошибка — масштабировать до проверки. У вас это как?»

**М:** F2: правило до прогона; карточка не в done без §5.

**Ж:** «ИИИ — пилот. Три реальные задачи, через месяц цифры. Письмо короткое, по делу.»

**Приоритеты для письма:** F6, F5, пилот ИИИ, F2.
