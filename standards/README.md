# Стандарты отчётов KOI (`standards/`)

Правила для людей и LLM, которые пишут отчёты по экспериментам в `projects/<id>/reports/`.

Отчёт привязан к карточке канбана; путь и индекс — в [docs/human/project-format.md](../docs/human/project-format.md).

## Файлы

| Файл | Назначение |
|------|------------|
| [templates/report-skeleton.md](templates/report-skeleton.md) | Шаблон отчёта: разделы §1–§N и блоки HOW-TODO под каждым пунктом |
| [no-ai-report-rules.md](no-ai-report-rules.md) | Что делать / не делать; публичные имена датасетов для текста статьи |

## Как применять

Человек: скопировать [templates/report-skeleton.md](templates/report-skeleton.md) в новый `.md`, удалить блоки `> HOW-TODO`, держать под рукой [no-ai-report-rules.md](no-ai-report-rules.md).

Агент / LLM: перед правкой прочитать оба файла; не менять `project.md`, только содержимое отчёта. Сохранять якоря разделов, ссылки на соседние карточки и пути к артефактам.

## Принципы

1. Один отчёт = одна карточка канбана; внутри — несколько прогонов (мини / большой / повтор).
2. Зачем (§1) → метрика (§2) → постановка, SMART-подзадачи и сбор (§3) → результаты и выводы (§4+).
3. Финальный тест (§2) отделён от диагностики в таблицах §4+.
4. В §3 — публичное имя + локальный путь; в выводах и §1 — только имя для статьи.
